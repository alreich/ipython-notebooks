Bayesian Data Analysis¶

This notebook provides a very simple example of Bayesian parameter estimation using the Beta-Binomial model. Both analytical and simulation-based results are presented.

Approaches to Data Analysis¶

"All models are wrong, but some are useful" -- George Box

1. Exploratory¶

Descriptive statistics without any probabilistic assumptions

Examples include: Average, Median, Quantiles, Range, Variance, Minimum/Maximum, Histogram, and various other plots and charts

[Note: Quantities in items 2 & 3, below, can be scalars or vectors.]

2. Frequentist¶

Assumes a probability model with a fixed, deterministic parameter, $\theta$

$X \sim F(x;\theta)$

e.g., $F(x;\theta) = \Phi(\frac{x-\mu}{\sigma})$, where $\theta = \begin{pmatrix} \mu \\ \sigma \end{pmatrix}$ and $\Phi^{\prime}(z) = \phi(z) = \frac{1}{\sqrt {2\pi}} e^{z^2/2}$

3. Bayesian¶

Assumes a probability model with a random parameter, $\Theta$, that has its own model

$(X \mid \Theta=\theta) \sim F(x;\theta)$ called the "Likelihood Distribution"

$\Theta \sim G(\theta;\gamma)$ called the "Prior Distribution"

$\gamma$ is called a "hyperparameter" and is usually deterministic

The next section provides an example.

To make inferences about a model, Frequentist or Bayesian, it is necessary to estimate the parameter, $\theta$, (scalar or vector) of the model.

In the Frequentist case, the Maximum Likelihood Estimate (MLE) is typically derived. The MLE is a deterministic value.

In the Bayesian case, the conditional probability distribution (or density) of the parameter, given the observed data, is usually derived: $P(\Theta \mid X=x)$. It's called the posterior distribution/density. Depending on the type of likelihood and prior, the derivation of the posterior might be intractable, so simulation is required, typically via a Markov Chain Monte Carlo (MCMC) algorithm.

Beta-Binomial Example¶

This is probably one of the most often sited examples of Bayesian estimation. The posterior distribution can be computed analytically. Also, the prior and posterior are from the same family of distributions, and so they are called conjugate distributions.

The Data¶

Assume that we've conducted an experiment consisting of $n$ binomial trials with an unknown probability of success, $\theta$, and that we've observed $k_{obs}$ successes:

In [1]:
n = 20  # Number of trials
k_obs = 6  # Number of observed successes in n trials

The Frequentist Binomial Model¶

Here, the parameter, $\theta$, is an unknown deterministic value.

$$K \sim Binomial_n(k;\theta) \equiv \binom{n}{k} \theta^k(1-\theta)^{n-k}$$
In [2]:
print(f"The Frequentists' MLE estimate of theta is k_obs/n = {k_obs}/{n} = {k_obs/n}")

The Frequentists' MLE estimate of theta is k_obs/n = 6/20 = 0.3

The Bayesian Beta-Binomial Model¶

Likelihood:¶

Probabilistic model of the data

$(K \mid \Theta=\theta) \sim Binomial_n(k;\theta) \equiv \binom{n}{k} \theta^k(1-\theta)^{n-k}$

Prior:¶

Probabilistic model of the likelihood model parameter

$\Theta \sim Beta(\theta; \alpha, \beta) \equiv \frac{\Gamma(\alpha)\Gamma(\beta)}{\Gamma(\alpha + \beta)} \theta^{\alpha - 1} (1-\theta)^{\beta - 1}$

For this example, assume $\Theta \sim Beta(1, 1)$, which is equivalent to the uniform distribution:

$Beta(1,1) \equiv U(0,1)$

In [3]:
# Assumed for this example
pre_alpha = 1
pre_beta = 1

Posterior¶

Probabilistic model of the parameter(s), given the observed data

According to Bayes Theorem, Posterior $\propto$ Likelihood $\times$ Prior

Note how similar the Binomial and Beta distribution are. So, after some algebraic manipulation we have,

$(\Theta \mid K=k) \sim Beta(\theta; \alpha+k, \beta+n-k)$

In [4]:
post_alpha = pre_alpha + k_obs
post_beta = pre_beta + n - k_obs
posterior_beta_parameters = (post_alpha, post_beta)
show_calc = f" = Beta({pre_alpha} + {k_obs}, {pre_beta} + {n} - {k_obs})"
print(f"So, the posterior distribution is Beta{posterior_beta_parameters}" + show_calc)

So, the posterior distribution is Beta(7, 15) = Beta(1 + 6, 1 + 20 - 6)

Plot of the Posterior Density¶

from scipy.stats import uniform, binom, beta
import numpy as np
import matplotlib.pyplot as plt

# Frequentist Confidence Interval
conf_int_size = 0.95

ci_low, ci_high = beta.ppf([p1, 1-p1], *posterior_beta_parameters)

_ = plt.plot(x, beta.pdf(x, *posterior_beta_parameters), 'r-', lw=3, alpha=0.6)

A Simple Simulation-Based Derivation of the Posterior¶

"We could call Bayesian data analysis 'statistics using conditional probability', but that wouldnt put the butts in the seats."

-- Andrew Gelman

This section illustrates how the beta-binomial posterior density can be derived, instead, by simulation.

The approach here is very simple for pedagogical purposes; MCMC is not used.

To do this, we'll work with two lists of $\theta$'s:

  • priors: a list of $m$ independent Uniform(0,1) random numbers

  • posteriors: the subset of priors that, when used in a $Binomial_n(k;\theta)$ trial, result in $k_{obs}$ successes

In [6]:
m = 500_000  # Number of priors

priors = uniform.rvs(size=m)
%time posteriors = [theta for theta in priors if binom.rvs(n, theta)==k_obs]

print(f"\nThere were {len(posteriors)} matches out of {m} priors. (Only {100*len(posteriors)/m:0.2}% matched)")

MCMC-Based Derivation of the Posterior¶

The type of simple simulation used above is not practical for most situations. Fortunately, there is a practical family of simulation algorithms known as Markov Chain Monte Carlo (MCMC) methods. And there are probabilistic programming libraries for applying MCMC methods (e.g., PyMC3).

Here's how PyMC3 can be used to solve the problem presented above.

In [8]:
import pymc3 as pm
import warnings
warnings.simplefilter('ignore')

Create the Model¶

In [9]:
model = pm.Model()

with model:
    theta = pm.Beta("theta", alpha=pre_alpha, beta=pre_beta )
    k = pm.Binomial("k", p=theta, n=n, observed=k_obs)

Credible Interval¶

The credible interval is for Bayesians what the confidence interval is for Frequentists.

The dark horizontal bar in the plot, below, is the highest posterior density (HPD), which is the minimum width credible interval for a specific requested size.
